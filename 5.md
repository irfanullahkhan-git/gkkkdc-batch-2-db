## 1. CASE WHEN - Conditional Column

`CASE WHEN` evaluates a condition for each row and returns a value based on which branch matches.

### Basic CASE WHEN (no alias)

```sql
SELECT Name, Marks, CASE WHEN Marks > 80 THEN "Excellent" ELSE "Good" END FROM students;
```

| Name | Marks | CASE WHEN Marks > 80 THEN "Excellent" ELSE "Good" END |
|---|---|---|
| Asma | 85 | Excellent |
| Bisma | 72 | Good |
| Bushra | 91 | Excellent |
| Bilal | 65 | Good |
| Daniyal | 78 | Good |
| Ali | 91 | Excellent |
| Gul | NULL | Good |
| Hina | 85 | Excellent |
| Iqra | 55 | Good |

Without an alias, the entire `CASE` expression becomes the column header — which is verbose. Gul's NULL marks fall into the `ELSE "Good"` branch.

### CASE WHEN with AS alias

```sql
SELECT Name, Marks, CASE WHEN Marks > 80 THEN "Excellent" ELSE "Good" END AS Remarks FROM students;
```

| Name | Marks | Remarks |
|---|---|---|
| Asma | 85 | Excellent |
| Bisma | 72 | Good |
| Bushra | 91 | Excellent |
| Bilal | 65 | Good |
| Daniyal | 78 | Good |
| Ali | 91 | Excellent |
| Gul | NULL | Good |
| Hina | 85 | Excellent |
| Iqra | 55 | Good |

`AS Remarks` gives the computed column a clean, readable name.

## 2. Nested CASE WHEN

A `CASE WHEN` expression can be nested inside another to handle multiple grade levels.

### First attempt (wrong syntax — missing WHEN)

```sql
SELECT Name, Marks, CASE WHEN Marks > 80 THEN "Excellent" ELSE CASE Marks > 50 THEN "Good" ELSE "Fail" END END AS Remarks FROM students;
```

```
ERROR 1064 (42000): You have an error in your SQL syntax
```

The inner `CASE` is missing the `WHEN` keyword. In MariaDB, every `CASE` must use `CASE WHEN condition THEN ...`.

### Second attempt (wrong — missing END for inner CASE)

```sql
SELECT Name, Marks, CASE WHEN Marks > 80 THEN "Excellent" ELSE CASE WHEN Marks > 50 THEN "Good" ELSE "Fail" END AS Remarks FROM students;
```

```
ERROR 1064 (42000): You have an error in your SQL syntax
```

The inner `CASE` is missing its own closing `END`. Every `CASE` expression needs a matching `END`.

### Correct nested CASE WHEN

```sql
SELECT Name, Marks, CASE WHEN Marks > 80 THEN "Excellent" ELSE CASE WHEN Marks > 50 THEN "Good" ELSE "Fail" END END AS Remarks FROM students;
```

| Name | Marks | Remarks |
|---|---|---|
| Asma | 85 | Excellent |
| Bisma | 72 | Good |
| Bushra | 91 | Excellent |
| Bilal | 65 | Good |
| Daniyal | 78 | Good |
| Ali | 91 | Excellent |
| Gul | NULL | Fail |
| Hina | 85 | Excellent |
| Iqra | 55 | Good |

Two `END` keywords close the two `CASE` expressions. Gul's NULL marks now correctly fall into `"Fail"` (NULL is not > 50).

## 3. Aggregate Functions

Aggregate functions operate on a **set of rows** and return a single summary value.

### COUNT(*) — Total row count

```sql
SELECT COUNT(*) FROM students;
```

| COUNT(*) |
|---|
| 9 |

Counts all rows regardless of NULL values.

### COUNT(*) with WHERE — Filtered count

```sql
SELECT COUNT(*) FROM students WHERE Address = "Islamabad";
```

| COUNT(*) |
|---|
| 3 |

Only 3 students live in Islamabad (Bushra, Bilal, Iqra).

```sql
SELECT * FROM students WHERE Address = "Islamabad";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 9 | Iqra | NULL | Islamabad | 55 |

### Mixing SELECT * with COUNT(*) (quirk)

```sql
SELECT *, COUNT(*) FROM students WHERE Address = "Islamabad";
```

| ID | Name | Age | Address | Marks | COUNT(*) |
|---|---|---|---|---|---|
| 3 | Bushra | 21 | Islamabad | 91 | 3 |

MariaDB returns only one row (the first matching row) alongside the aggregate count. This is a non-standard MariaDB behaviour — mixing row columns with aggregates without `GROUP BY` is generally avoided.

### SUM — Sum of all values

```sql
SELECT SUM(Marks) FROM students;
```

| SUM(Marks) |
|---|
| 622 |

`SUM` ignores NULL values. Gul's NULL marks are not included in the total.

> **Note:** `SUM(10, 20)` is a syntax error — `SUM` takes a single column or expression, not a comma-separated list.

### AVG — Average

```sql
SELECT AVG(Marks) FROM students;
```

| AVG(Marks) |
|---|
| 77.7500 |

Divides the sum (622) by the count of non-NULL marks (8 rows). Gul's NULL is excluded from both numerator and denominator.

### MAX — Highest value

```sql
SELECT MAX(Marks) FROM students;
```

| MAX(Marks) |
|---|
| 91 |

### MIN — Lowest value

```sql
SELECT MIN(Marks) FROM students;
```

| MIN(Marks) |
|---|
| 55 |

## 4. Math Functions

### CEIL — Round up to nearest integer

```sql
SELECT CEIL(AVG(Marks)) FROM students;
```

| CEIL(AVG(Marks)) |
|---|
| 78 |

`AVG` returns 77.75. `CEIL` rounds it **up** to 78 regardless of the decimal.

```sql
SELECT CEIL(2.6);
```

| CEIL(2.6) |
|---|
| 3 |

```sql
SELECT CEIL(3);
```

| CEIL(3) |
|---|
| 3 |

An integer is already a whole number — `CEIL` returns it unchanged.

```sql
SELECT CEIL(2.00000001);
```

| CEIL(2.00000001) |
|---|
| 3 |

Even a tiny fraction above 2 causes `CEIL` to return 3.

### FLOOR — Round down to nearest integer

```sql
SELECT FLOOR(AVG(Marks)) FROM students;
```

| FLOOR(AVG(Marks)) |
|---|
| 77 |

`FLOOR` rounds **down**, so 77.75 becomes 77.

```sql
SELECT FLOOR(2.00000001);
```

| FLOOR(2.00000001) |
|---|
| 2 |

Any value between 2 and 3 (exclusive) floors to 2.

> **Calling with a table:** `SELECT CEIL(2.6) FROM students;` returns 9 rows, all with value 3 — one row per student. `SELECT CEIL(2.6)` (without FROM) returns a single row.

### ROUND — Round to nearest integer or decimal place

```sql
SELECT ROUND(2.543);
```

| ROUND(2.543) |
|---|
| 3 |

.543 ≥ .5, so it rounds up.

```sql
SELECT ROUND(2.443);
```

| ROUND(2.443) |
|---|
| 2 |

.443 < .5, so it rounds down.

### ROUND with decimal places

```sql
SELECT ROUND(2.443, 2);
```

| ROUND(2.443, 2) |
|---|
| 2.44 |

```sql
SELECT ROUND(2.446, 2);
```

| ROUND(2.446, 2) |
|---|
| 2.45 |

The second argument specifies how many decimal places to keep. Standard rounding rules apply at that position.

### ABS — Absolute value

```sql
SELECT ABS(-5);
```

| ABS(-5) |
|---|
| 5 |

```sql
SELECT ABS(5-100);
```

| ABS(5-100) |
|---|
| 95 |

`ABS` returns the non-negative magnitude of a value.

### MOD — Remainder (modulo)

```sql
SELECT MOD(10, 3);
```

| MOD(10, 3) |
|---|
| 1 |

10 ÷ 3 = 3 remainder **1**.

```sql
SELECT MOD(3, 10);
```

| MOD(3, 10) |
|---|
| 3 |

3 ÷ 10 = 0 remainder **3**. When the dividend is smaller than the divisor, the remainder equals the dividend.

### POW — Exponentiation

```sql
SELECT POW(2,3);
```

| POW(2,3) |
|---|
| 8 |

2³ = 8.

```sql
SELECT POW(2,-3);
```

| POW(2,-3) |
|---|
| 0.125 |

2⁻³ = 1/8 = 0.125. Negative exponents work and return a decimal.

## Summary

| Concept | Syntax | What it does |
|---|---|---|
| **CASE WHEN** | `CASE WHEN cond THEN val ELSE val END` | Conditional value per row |
| **AS alias** | `... END AS ColName` | Names the computed column |
| **Nested CASE** | `ELSE CASE WHEN ... END END` | Multi-level branching; each CASE needs its own END |
| **COUNT(*)** | `SELECT COUNT(*) FROM t` | Counts all rows (including NULLs) |
| **SUM** | `SELECT SUM(col) FROM t` | Sum of non-NULL values |
| **AVG** | `SELECT AVG(col) FROM t` | Average of non-NULL values |
| **MAX / MIN** | `SELECT MAX(col), MIN(col) FROM t` | Highest / lowest non-NULL value |
| **CEIL** | `CEIL(x)` | Round up to nearest integer |
| **FLOOR** | `FLOOR(x)` | Round down to nearest integer |
| **ROUND** | `ROUND(x)` / `ROUND(x, n)` | Round to nearest integer or n decimal places |
| **ABS** | `ABS(x)` | Absolute (non-negative) value |
| **MOD** | `MOD(x, y)` | Remainder of x ÷ y |
| **POW** | `POW(x, y)` | x raised to the power y |

### Errors Encountered

| Query | Error | Reason |
|---|---|---|
| `SELECT SUM(10, 20)` | Syntax error | `SUM` takes a single argument, not a list |
| `ELSE CASE Marks > 50 THEN ...` | Syntax error | Inner `CASE` missing `WHEN` keyword |
| `ELSE CASE WHEN ... END AS Remarks` | Syntax error | Inner `CASE` missing its closing `END` |
