## 1. DISTINCT — Remove Duplicates

```sql
SELECT DISTINCT Address FROM students;
```

Returns each unique city only once (Rawalpindi, Lahore, Islamabad, Peshawar, Nowshera).

---

## 2. LIMIT and OFFSET — Pagination

### Get first N rows

```sql
SELECT * FROM students LIMIT 2;
```

Returns rows 1–2.

### Skip rows then get N

```sql
SELECT * FROM students LIMIT 2 OFFSET 1;
-- Returns rows 2–3 (skips the first 1 row)

SELECT * FROM students LIMIT 2 OFFSET 2;
-- Returns rows 3–4 (skips the first 2 rows)

SELECT * FROM students LIMIT 3 OFFSET 4;
-- Returns rows 5–7 (skips the first 4 rows)
```

> `OFFSET` cannot be used alone — it requires `LIMIT`.
> Negative values in `LIMIT` are not allowed.

## 3. LIKE — Pattern Matching

`LIKE` searches for a pattern in a text column.

### Wildcards

| Wildcard | Meaning | Example |
|---|---|---|
| `%` | Zero or more characters | `"B%"` matches Bisma, Bushra, Bilal |
| `_` | Exactly one character | `"___"` matches any 3-letter name |

### Examples

```sql
-- Names starting with "B"
SELECT * FROM students WHERE Name LIKE "B%";

-- Names starting with "Bi"
SELECT * FROM students WHERE Name LIKE "Bi%";

-- Names containing "i" anywhere
SELECT * FROM students WHERE Name LIKE "%i%";

-- Names ending with "l"
SELECT * FROM students WHERE Name LIKE "%l";

-- Exactly 3 characters long
SELECT * FROM students WHERE Name LIKE "___";
```

### Combining LIKE with OR / NOT

```sql
-- Names ending with "i" OR starting with "i"
SELECT * FROM students WHERE Name LIKE "%i" OR Name LIKE "i%";

-- Names that do NOT end or start with "i"
SELECT * FROM students WHERE NOT (Name LIKE "%i" OR Name LIKE "i%");

-- Contains "i" but does not start or end with "i"
SELECT * FROM students
WHERE Name LIKE "%i%"
  AND NOT (Name LIKE "%i" OR Name LIKE "i%");
```

> `LIKE` is case-insensitive by default in MariaDB/MySQL.

## 4. BETWEEN — Range Filtering

`BETWEEN` is **inclusive** on both ends.

```sql
-- Students aged 20 to 22
SELECT * FROM students WHERE Age BETWEEN 20 AND 22;

-- Alphabetical range (names starting A–C)
SELECT * FROM students WHERE Name BETWEEN "A" AND "D";
```

---

## 5. IN — Match Against a List

```sql
-- Equivalent to Age = 20 OR Age = 21 OR Age = 23
SELECT * FROM students WHERE Age IN (20, 21, 23);

-- NOT IN
SELECT * FROM students WHERE Age NOT IN (20, 21, 23);
```

`IN` is cleaner than chaining multiple `OR` conditions.

---

## 6. IS NULL / IS NOT NULL

`NULL` cannot be compared with `=` — you must use `IS NULL` or `IS NOT NULL`.

```sql
-- Students with no marks recorded
SELECT * FROM students WHERE Marks IS NULL;

-- Students with marks recorded
SELECT * FROM students WHERE Marks IS NOT NULL;

-- Students with no 3-letter name (NOT LIKE with underscores)
SELECT * FROM students WHERE Name NOT LIKE "___";
```

---

## Key Points

| Concept | Syntax |
|---|---|
| Unique values | `SELECT DISTINCT col FROM t` |
| Limit rows | `LIMIT n` |
| Skip + limit | `LIMIT n OFFSET m` |
| Any chars wildcard | `LIKE "%pattern%"` |
| Single char wildcard | `LIKE "___"` (one `_` per character) |
| Inclusive range | `col BETWEEN a AND b` |
| Match list | `col IN (v1, v2, v3)` |
| Check NULL | `col IS NULL` / `col IS NOT NULL` |
