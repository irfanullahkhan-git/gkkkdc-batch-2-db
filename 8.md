## 1. CASE WHEN — Conditional Logic

`CASE` works like an if-else expression inside a query.

### Simple two-branch CASE

```sql
SELECT Name, Marks,
    CASE WHEN Marks > 80 THEN "Excellent" ELSE "Good" END AS Remarks
FROM students;
```

| Name | Marks | Remarks |
|---|---|---|
| Asma | 85 | Excellent |
| Bisma | 72 | Good |
| Bushra | 91 | Excellent |
| Bilal | 65 | Good |
| Daniyal | 78 | Good |
| Ali | 91 | Excellent |
| Gul | NULL | Good |
| Hina | 85 | Excellent |
| Iqra | 55 | Good |

The column header is set with `AS Remarks`.

### Nested CASE

```sql
SELECT Name, Marks,
    CASE
        WHEN Marks > 80 THEN "Excellent"
        ELSE
            CASE
                WHEN Marks > 50 THEN "Good"
                ELSE "Fail"
            END
    END AS Remarks
FROM students;
```

Each inner `CASE` must have its own `END`. `Gul` (NULL marks) falls through to "Fail".

> The syntax `CASE Marks > 50 THEN …` (without `WHEN`) is invalid — always use `CASE WHEN condition`.

## 2. Aggregate Functions

Aggregate functions compute a single result from multiple rows. They **ignore NULL** values (except `COUNT(*)`).

| Function | Description |
|---|---|
| `COUNT(*)` | Total number of rows |
| `COUNT(col)` | Rows where col is NOT NULL |
| `SUM(col)` | Sum of all values |
| `AVG(col)` | Average (NULLs excluded) |
| `MAX(col)` | Largest value |
| `MIN(col)` | Smallest value |

### Examples

```sql
SELECT COUNT(*) FROM students;
-- 9

SELECT COUNT(*) FROM students WHERE Address = "Islamabad";
-- 3

SELECT SUM(Marks) FROM students;
-- 622

SELECT AVG(Marks) FROM students;
-- 77.7500  (622 / 8, because Gul's NULL is excluded)

SELECT MAX(Marks) FROM students;
-- 91

SELECT MIN(Marks) FROM students;
-- 55
```

> Mixing a non-aggregated column with an aggregate without `GROUP BY` returns one arbitrary row:
> `SELECT *, COUNT(*) FROM students WHERE Address = "Islamabad"` → returns 1 row with COUNT = 3.

## 3. Math Functions

### Rounding

```sql
SELECT CEIL(AVG(Marks))  FROM students;   -- 78  (rounds up)
SELECT FLOOR(AVG(Marks)) FROM students;   -- 77  (rounds down)

SELECT CEIL(2.6);            -- 3
SELECT FLOOR(2.00000001);    -- 2
SELECT ROUND(2.543);         -- 3   (rounds to nearest integer)
SELECT ROUND(2.443);         -- 2
SELECT ROUND(2.443, 2);      -- 2.44
SELECT ROUND(2.446, 2);      -- 2.45
```

> `CEIL(n)` always rounds **up** — even `CEIL(2.00000001)` returns 3.

### Other math functions

```sql
SELECT ABS(-5);        -- 5   (absolute value)
SELECT ABS(5-100);     -- 95

SELECT MOD(10, 3);     -- 1   (remainder of 10 ÷ 3)
SELECT MOD(3, 10);     -- 3   (remainder of 3 ÷ 10)

SELECT POW(2, 3);      -- 8   (2³)
SELECT POW(2, -3);     -- 0.125  (2⁻³ = 1/8)
```

### Running a scalar function against a table

```sql
SELECT CEIL(2.6) FROM students;
```

This returns 9 rows (one per student) all showing `3`. To get a single value, omit the `FROM`:

```sql
SELECT CEIL(2.6);   -- 1 row: 3
```

## Key Points

| Concept | Notes |
|---|---|
| `CASE WHEN … THEN … ELSE … END` | Conditional expression; must close every `CASE` with `END` |
| `COUNT(*)` | Counts all rows including NULLs |
| `COUNT(col)` | Counts only non-NULL values in col |
| `AVG`, `SUM`, `MAX`, `MIN` | All ignore NULL values |
| `CEIL(n)` | Rounds up to next integer |
| `FLOOR(n)` | Rounds down to previous integer |
| `ROUND(n, d)` | Rounds to *d* decimal places |
| `ABS(n)` | Removes the sign |
| `MOD(a, b)` | Remainder of a ÷ b |
| `POW(a, b)` | a raised to the power b |
