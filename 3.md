## Table Setup

The old `students` table was dropped and a new one was created with additional columns.

### First attempt (failed)

```sql
CREATE TABLE students (
    ID INT NOT NULL AUTO_INCREMENT,
    Name VARCHAR(30),
    Age INT,
    Address VARCHAR(100),
    Marks INT
);
```

```
ERROR 1075 (42000): Incorrect table definition;
there can be only one auto column and it must be defined as a key
```

`AUTO_INCREMENT` requires the column to be a **key** (UNIQUE or PRIMARY KEY). Without it, MariaDB rejects the table.

### Correct version

```sql
CREATE TABLE students (
    ID INT NOT NULL UNIQUE AUTO_INCREMENT,
    Name VARCHAR(30),
    Age INT,
    Address VARCHAR(100),
    Marks INT
);
```

### Table structure

| Field | Type | Null | Key | Default | Extra |
|---|---|---|---|---|---|
| ID | int(11) | NO | PRI | NULL | auto_increment |
| Name | varchar(30) | YES | | NULL | |
| Age | int(11) | YES | | NULL | |
| Address | varchar(100) | YES | | NULL | |

```sql
SELECT * FROM students ORDER BY Name;
```

## ORDER BY Multiple Columns

When two rows have the **same value** in the first sort column, the second column is used to break the tie.

```sql
SELECT * FROM students ORDER BY Marks DESC, Age DESC;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 6 | Ali | **22** | Peshawar | **91** |
| 3 | Bushra | **21** | Islamabad | **91** |
| 1 | Asma | 20 | Rawalpindi | 85 |
| 8 | Hina | 20 | Lahore | 85 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 2 | Bisma | 22 | Lahore | 72 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 9 | Iqra | NULL | Islamabad | 55 |
| 7 | Gul | 24 | Nowshera | NULL |

Both Ali and Bushra have Marks = 91. Since we also sort by `Age DESC`, Ali (age 22) comes **before** Bushra (age 21).

Similarly, Asma and Hina both have Marks = 85 and the same Age = 20, so their order is determined by their original row order.

## ORDER BY Column Position (Number)

Instead of writing the column name, you can use the **position number** of the column in the SELECT list.

### Position 1 = first column in SELECT

```sql
SELECT * FROM students ORDER BY 1 DESC;
```

Here `1` refers to the first column — `ID`. Result is sorted by ID descending (9, 8, 7, ... 1).

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 9 | Iqra | NULL | Islamabad | 55 |
| 8 | Hina | 20 | Lahore | 85 |
| 7 | Gul | 24 | Nowshera | NULL |
| 6 | Ali | 22 | Peshawar | 91 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 2 | Bisma | 22 | Lahore | 72 |
| 1 | Asma | 20 | Rawalpindi | 85 |

### Position 2 = second column (Name)

```sql
SELECT * FROM students ORDER BY 2 DESC;
```

`2` refers to `Name` — result is sorted by Name descending (Z to A).

### Position depends on the SELECT list, not the table

```sql
SELECT Name, Age FROM students ORDER BY 2 DESC;
```

| Name | Age |
|---|---|
| Gul | 24 |
| Bilal | 23 |
| Bisma | 22 |
| Ali | 22 |
| Bushra | 21 |
| Asma | 20 |
| Daniyal | 20 |
| Hina | 20 |
| Iqra | NULL |

Here `2` refers to `Age` (the 2nd column in `SELECT Name, Age`), **not** `Bisma` or any table column.

> `NULL` values appear **last** in `DESC` order.

### Invalid column position (error)

```sql
SELECT Name, Age FROM students ORDER BY 20 DESC;
```

```
ERROR 1054 (42S22): Unknown column '20' in 'order clause'
```

```sql
SELECT Name, Age FROM students ORDER BY 3 DESC;
```

```
ERROR 1054 (42S22): Unknown column '3' in 'order clause'
```

The SELECT list only has 2 columns (Name, Age), so positions `3` and `20` do not exist.

## LIMIT - Restricting Number of Rows

### Get top 3 students by marks

```sql
SELECT * FROM students ORDER BY Marks DESC LIMIT 3;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 6 | Ali | 22 | Peshawar | 91 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 1 | Asma | 20 | Rawalpindi | 85 |

`ORDER BY Marks DESC` sorts highest marks first, then `LIMIT 3` keeps only the first 3 rows from that sorted result.

> **Tip:** `LIMIT` without `ORDER BY` returns an unpredictable set of rows. Always pair them together when you want meaningful top-N results.

---

## Summary

| Concept | Syntax | Notes |
|---|---|---|
| Sort ascending | `ORDER BY col ASC` | `ASC` is the default, can be omitted |
| Sort descending | `ORDER BY col DESC` | Must be written explicitly |
| Sort by multiple columns | `ORDER BY col1 DESC, col2 ASC` | Second column breaks ties in first |
| Sort by column position | `ORDER BY 2 DESC` | Position is based on the SELECT list |
| Limit results | `LIMIT n` | Returns first `n` rows |
| Top-N query | `ORDER BY col DESC LIMIT n` | Common pattern for "top N" results |
| NULL in ORDER BY | — | NULLs appear first in ASC, last in DESC |
| AUTO_INCREMENT rule | Must have `UNIQUE` or `PRIMARY KEY` | Otherwise table creation fails |
