# String Functions and GROUP BY

> Continues from [8.md](8.md). Uses the `college.students` table.

---

## 1. String Functions

### UPPER / LOWER

```sql
SELECT UPPER(Name) FROM students;   -- ASMA, BISMA, ...
SELECT LOWER(Name) FROM students;   -- asma, bisma, ...
```

### CONCAT — Join Strings

```sql
SELECT CONCAT(Name, " is a student and lives in ", Address) FROM students;
```

> If **any** argument is `NULL`, `CONCAT` returns `NULL` for that row.
> That is why Gul's row (NULL Marks) returns NULL when Marks is included.

```sql
-- NULL propagation example
SELECT CONCAT("Irfan", " Khan", NULL);   -- NULL
```

### LENGTH — Character Count

```sql
SELECT Name, LENGTH(Name) FROM students;
```

| Name | LENGTH(Name) |
|---|---|
| Asma | 4 |
| Bisma | 5 |
| Bushra | 6 |
| Bilal | 5 |
| Daniyal | 7 |
| Ali | 3 |
| Gul | 3 |
| Hina | 4 |
| Iqra | 4 |

`LENGTH` also works on numbers (returns the digit count as a string):

```sql
SELECT LENGTH(Marks) FROM students;   -- 2 for all 2-digit marks; NULL for Gul
```

### SUBSTRING — Extract Part of a String

```sql
-- SUBSTRING(string, start_position, length)
-- Positions are 1-indexed (position 0 returns empty string)

SELECT SUBSTRING(Name, 1, 2) FROM students;   -- First 2 chars: As, Bi, Bu, ...
SELECT SUBSTRING(Name, 1, 3) FROM students;   -- First 3 chars: Asm, Bis, Bus, ...

-- Negative position counts from the end
SELECT SUBSTRING(Name, -1, 3) FROM students;  -- Last char only (max length 3 from end): a, a, a, ...
SELECT SUBSTRING(Name, -3)    FROM students;  -- Last 3 chars: sma, sma, hra, ...
SELECT SUBSTRING(Name, -2)    FROM students;  -- Last 2 chars: ma, ma, ra, ...

-- Starting from position 2
SELECT SUBSTRING(Name, 2)     FROM students;  -- sma, isma, ushra, ...
SELECT SUBSTRING(Name, 2, 1)  FROM students;  -- Single char at position 2: s, i, u, ...
SELECT SUBSTRING(Name, 2, 10) FROM students;  -- Up to 10 chars from pos 2: sma, isma, ...
```

> Position `0` behaves like an invalid start and returns an empty string.
> A negative length returns an empty string.

### REPLACE — Substitute Characters

```sql
-- Replace lowercase 'a' with '@' (case-sensitive)
SELECT Name, REPLACE(Name, 'a', '@') FROM students;
-- Asma → Asm@   Bilal → Bil@l   Ali → Ali (no lowercase 'a')

-- Convert to lowercase first, then replace all 'a'
SELECT Name, REPLACE(LOWER(Name), 'a', '@') FROM students;
-- Asma → @sm@   Ali → @li
```

### REVERSE — Reverse a String

```sql
SELECT Name, REVERSE(Name) FROM students;
-- Asma → amsA   Daniyal → layinaD

SELECT Name, Marks, REVERSE(Marks) FROM students;
-- 85 → 58   91 → 19   NULL → NULL
```

### NULL Handling Functions

```sql
-- IFNULL(expr, fallback): return fallback when expr IS NULL
SELECT Name, IFNULL(Marks, 0) FROM students;
-- Gul's NULL Marks becomes 0

-- Combining IFNULL with REVERSE
SELECT Name, REVERSE(IFNULL(Marks, 0)) FROM students;
-- Gul → 0 (reversed "0")

-- NULLIF(a, b): return NULL if a = b, otherwise return a
SELECT REVERSE(NULLIF(Marks, 0)) FROM students;
-- Marks = 0 would become NULL before reversing (no zeros here, same result as REVERSE)
```

## 2. GROUP BY — Grouping Rows

`GROUP BY` groups rows with the same value in a column so aggregate functions can be applied per group.

### Count students per city

```sql
SELECT Address, COUNT(*) FROM students GROUP BY Address;
```

| Address | COUNT(*) |
|---|---|
| Islamabad | 3 |
| Lahore | 2 |
| Nowshera | 1 |
| Peshawar | 1 |
| Rawalpindi | 2 |

Without `GROUP BY`, mixing a column and `COUNT(*)` returns only one arbitrary row.

### Group by a single column

```sql
SELECT Age,   COUNT(*) FROM students GROUP BY Age;
SELECT Marks, COUNT(*) FROM students GROUP BY Marks;
```

NULL values form their own group.

### Group by multiple columns

```sql
SELECT Address, Age, COUNT(*) FROM students GROUP BY Address, Age;
```

Each unique *(Address, Age)* combination becomes one group.

### Combining GROUP BY with ORDER BY and LIMIT

```sql
-- Cities ordered alphabetically descending
SELECT Address, Age, COUNT(*) FROM students GROUP BY Address, Age ORDER BY 1 DESC;

-- Groups ordered by count
SELECT Address, Age, COUNT(*) AS total FROM students GROUP BY Address, Age ORDER BY 3 DESC;

-- Most common age
SELECT Age, COUNT(*) AS TotalStudents FROM students GROUP BY Age ORDER BY 2 DESC;

-- Only the most common age (top 1)
SELECT Age, COUNT(*) AS TotalStudents FROM students GROUP BY Age ORDER BY 2 DESC LIMIT 1;
```

### Subquery to extract a single grouped value

When you need just one column from a grouped subquery, every derived table must have an alias:

```sql
-- Most common age (scalar result)
SELECT StntAge
FROM (
    SELECT Age AS StntAge, COUNT(*) AS TotalStudents
    FROM students
    GROUP BY Age
    ORDER BY 2 DESC
    LIMIT 1
) AS Agr;
```

> **Rule:** Every column in `SELECT` that is not inside an aggregate function **must** appear in `GROUP BY`.

## Key Points

| Function | Purpose |
|---|---|
| `UPPER(s)` / `LOWER(s)` | Change case |
| `CONCAT(a, b, …)` | Join strings (NULL in any arg → NULL result) |
| `LENGTH(s)` | Character count |
| `SUBSTRING(s, pos, len)` | Extract substring (1-indexed; negative pos = from end) |
| `REPLACE(s, old, new)` | Substitute text (case-sensitive) |
| `REVERSE(s)` | Reverse string or number |
| `IFNULL(expr, val)` | Substitute NULL with a fallback value |
| `NULLIF(a, b)` | Return NULL when a equals b |
| `GROUP BY col` | Group rows for aggregate functions |
| Derived table alias | `(SELECT …) AS alias` — alias is required in MariaDB |
