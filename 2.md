## 1. DISTINCT - Remove Duplicate Values

`DISTINCT` returns only unique values from a column.

```sql
SELECT DISTINCT Address FROM students;
```

| Address |
|---|
| Rawalpindi |
| Lahore |
| Islamabad |
| Peshawar |
| Nowshera |

Without `DISTINCT`, "Rawalpindi", "Lahore", and "Islamabad" would each appear multiple times.

```sql
SELECT DISTINCT Age FROM students;
```

Returns each unique age only once (including `NULL`).

### DISTINCT on multiple columns

```sql
SELECT DISTINCT Age, Address FROM students;
```

Returns unique **combinations** of Age and Address together.

## 2. ORDER BY - Sorting Results

### Ascending (default)

```sql
SELECT * FROM students ORDER BY Name;
-- Same as:
SELECT * FROM students ORDER BY Name ASC;
```

### Descending

```sql
SELECT * FROM students ORDER BY Marks DESC;
```

### Sort by multiple columns

```sql
SELECT * FROM students ORDER BY Address ASC, Name ASC;
```

First sorts by Address alphabetically. If two students have the same address, it then sorts by Name.

### Sort by column position

```sql
SELECT Name, Marks FROM students ORDER BY 2 DESC;
```

`2` refers to the second column in the SELECT list (Marks).

> `NULL` values are sorted first in ascending order and last in descending order (in MySQL/MariaDB).

## 3. LIMIT and OFFSET - Controlling Result Size

### Get first N rows

```sql
SELECT * FROM students LIMIT 3;
```

Returns only the first 3 rows.

### Skip rows then get N (pagination)

```sql
SELECT * FROM students LIMIT 3 OFFSET 2;
```

Skips the first 2 rows, then returns the next 3.

### Shorthand syntax

```sql
SELECT * FROM students LIMIT 2, 3;
-- Same as LIMIT 3 OFFSET 2
-- Format: LIMIT offset, count
```

> **Tip:** `ORDER BY` + `LIMIT` together is very common. Without `ORDER BY`, the rows returned by `LIMIT` are unpredictable.

### Get top 3 highest marks

```sql
SELECT Name, Marks FROM students ORDER BY Marks DESC LIMIT 3;
```

## 4. LIKE - Pattern Matching

`LIKE` is used in `WHERE` to search for patterns in text columns.

### Wildcards

| Wildcard | Meaning | Example |
|---|---|---|
| `%` | Zero or more characters | `"A%"` matches Ali, Asma, Akber |
| `_` | Exactly one character | `"_i%"` matches Bisma, Bilal, Hina |

### Names starting with "B"

```sql
SELECT * FROM students WHERE Name LIKE "B%";
```

### Names ending with "a"

```sql
SELECT * FROM students WHERE Name LIKE "%a";
```

### Names containing "sh"

```sql
SELECT * FROM students WHERE Name LIKE "%sh%";
```

### Names that are exactly 4 characters long

```sql
SELECT * FROM students WHERE Name LIKE "____";
```

(Four underscores)

### NOT LIKE

```sql
SELECT * FROM students WHERE Name NOT LIKE "B%";
```

Returns everyone whose name does **not** start with B.

> `LIKE` is **case-insensitive** by default in MySQL/MariaDB.

## 5. BETWEEN - Range Filtering

`BETWEEN` checks if a value falls within a range (**inclusive** on both ends).

```sql
SELECT * FROM students WHERE Marks BETWEEN 70 AND 90;
```

This is equivalent to:

```sql
SELECT * FROM students WHERE Marks >= 70 AND Marks <= 90;
```

### NOT BETWEEN

```sql
SELECT * FROM students WHERE Marks NOT BETWEEN 70 AND 90;
```

Returns students with marks below 70 or above 90.

### BETWEEN with text (alphabetical range)

```sql
SELECT * FROM students WHERE Name BETWEEN "A" AND "D";
```

Returns names starting with A, B, or C (alphabetical comparison).

---

## 6. IN - Matching Against a List

`IN` checks if a value matches **any** value in a given list.

```sql
SELECT * FROM students WHERE Address IN ("Islamabad", "Lahore", "Peshawar");
```

This is equivalent to:

```sql
SELECT * FROM students
WHERE Address = "Islamabad"
   OR Address = "Lahore"
   OR Address = "Peshawar";
```

`IN` is cleaner and easier to read than chaining multiple `OR`s.

### NOT IN

```sql
SELECT * FROM students WHERE Address NOT IN ("Islamabad", "Lahore");
```

## 7. IS NULL / IS NOT NULL

`NULL` is special - you **cannot** compare it with `=` or `!=`. You must use `IS NULL` or `IS NOT NULL`.

```sql
-- WRONG (will not work as expected)
SELECT * FROM students WHERE Age = NULL;

-- CORRECT
SELECT * FROM students WHERE Age IS NULL;
```

### Find students with marks recorded

```sql
SELECT * FROM students WHERE Marks IS NOT NULL;
```

## 8. Aggregate Functions

Aggregate functions perform a calculation on a set of values and return a **single** value.

| Function | Description |
|---|---|
| `COUNT()` | Number of rows |
| `SUM()` | Total of a numeric column |
| `AVG()` | Average of a numeric column |
| `MIN()` | Smallest value |
| `MAX()` | Largest value |

### Examples

```sql
SELECT COUNT(*) FROM students;
-- 9 (total rows)

SELECT COUNT(Marks) FROM students;
-- 8 (NULL is not counted)

SELECT SUM(Marks) FROM students;
-- 522

SELECT AVG(Marks) FROM students;
-- 65.25 (522 / 8, NULLs ignored)

SELECT MIN(Marks) FROM students;
-- 55

SELECT MAX(Marks) FROM students;
-- 91
```

### With aliases

```sql
SELECT
    COUNT(*) AS "Total Students",
    AVG(Marks) AS "Average Marks",
    MAX(Marks) AS "Highest Marks",
    MIN(Marks) AS "Lowest Marks"
FROM students;
```

### COUNT with DISTINCT

```sql
SELECT COUNT(DISTINCT Address) FROM students;
-- 5 (unique cities only)
```

> **Key rule:** All aggregate functions **ignore NULL** values, except `COUNT(*)` which counts all rows regardless.

## 9. GROUP BY - Grouping Rows

`GROUP BY` groups rows that have the same value in a column, then you can apply aggregate functions to each group.

### Count students per city

```sql
SELECT Address, COUNT(*) AS "Total"
FROM students
GROUP BY Address;
```

| Address | Total |
|---|---|
| Rawalpindi | 2 |
| Lahore | 2 |
| Islamabad | 3 |
| Peshawar | 1 |
| Nowshera | 1 |

### Average marks per city

```sql
SELECT Address, AVG(Marks) AS "Avg Marks"
FROM students
GROUP BY Address;
```

### Group by multiple columns

```sql
SELECT Address, Age, COUNT(*) AS "Total"
FROM students
GROUP BY Address, Age;
```

> **Rule:** Every column in `SELECT` that is **not** inside an aggregate function **must** appear in `GROUP BY`.

## 10. HAVING - Filtering Groups

`HAVING` filters **after** grouping (like `WHERE` but for groups).

### Cities with more than 1 student

```sql
SELECT Address, COUNT(*) AS Total
FROM students
GROUP BY Address
HAVING Total > 1;
```

| Address | Total |
|---|---|
| Rawalpindi | 2 |
| Lahore | 2 |
| Islamabad | 3 |

### WHERE vs HAVING

| Feature | `WHERE` | `HAVING` |
|---|---|---|
| Filters | Individual rows | Groups (after `GROUP BY`) |
| Can use aggregate functions | No | Yes |
| Applied | Before grouping | After grouping |

### Using both together

```sql
SELECT Address, AVG(Marks) AS "Avg Marks"
FROM students
WHERE Marks IS NOT NULL
GROUP BY Address
HAVING AVG(Marks) > 75;
```

`WHERE` filters out NULL marks first, then groups by address, then `HAVING` keeps only groups with average > 75.

## 11. Common String Functions in SELECT

| Function | Description | Example | Result |
|---|---|---|---|
| `UPPER(str)` | Convert to uppercase | `UPPER("ali")` | `ALI` |
| `LOWER(str)` | Convert to lowercase | `LOWER("ALI")` | `ali` |
| `LENGTH(str)` | Number of characters | `LENGTH("Asma")` | `4` |
| `CONCAT(a, b)` | Join strings together | `CONCAT("Mr. ", Name)` | `Mr. Asma` |
| `SUBSTRING(str, pos, len)` | Extract part of a string | `SUBSTRING("Asma", 1, 2)` | `As` |
| `TRIM(str)` | Remove leading/trailing spaces | `TRIM("  Ali  ")` | `Ali` |
| `REPLACE(str, old, new)` | Replace text | `REPLACE("Ali", "A", "O")` | `Oli` |
| `REVERSE(str)` | Reverse the string | `REVERSE("Asma")` | `amsA` |

### Example: Full greeting column

```sql
SELECT CONCAT("Hello, ", UPPER(Name), "!") AS Greeting FROM students;
```

## 12. Numeric Functions

| Function | Description | Example | Result |
|---|---|---|---|
| `ROUND(n, d)` | Round to `d` decimal places | `ROUND(65.25, 1)` | `65.3` |
| `CEIL(n)` | Round up to nearest integer | `CEIL(65.25)` | `66` |
| `FLOOR(n)` | Round down to nearest integer | `FLOOR(65.75)` | `65` |
| `ABS(n)` | Absolute value | `ABS(-5)` | `5` |
| `MOD(a, b)` | Remainder of a / b | `MOD(10, 3)` | `1` |
| `POW(a, b)` | a raised to power b | `POW(2, 3)` | `8` |

### Example: Calculate grade percentage

```sql
SELECT Name, Marks, ROUND(Marks / 100.0 * 100, 1) AS Percentage FROM students;
```

## 13. CASE WHEN - Conditional Logic

`CASE` works like an if-else inside a query.

```sql
SELECT Name, Marks,
    CASE
        WHEN Marks >= 90 THEN "A+"
        WHEN Marks >= 80 THEN "A"
        WHEN Marks >= 70 THEN "B"
        WHEN Marks >= 60 THEN "C"
        WHEN Marks IS NULL THEN "N/A"
        ELSE "F"
    END AS Grade
FROM students;
```

| Name | Marks | Grade |
|---|---|---|
| Asma | 85 | A |
| Bisma | 72 | B |
| Bushra | 91 | A+ |
| Bilal | 65 | C |
| Daniyal | 78 | B |
| Fahad | 91 | A+ |
| Gul | NULL | N/A |
| Hina | 85 | A |
| Iqra | 55 | F |

### CASE inside aggregate

```sql
SELECT
    COUNT(CASE WHEN Marks >= 80 THEN 1 END) AS "Pass with A",
    COUNT(CASE WHEN Marks < 60 THEN 1 END) AS "Fail"
FROM students;
```

## 14. Subqueries in SELECT

A subquery is a query nested inside another query.

### Find students with the highest marks

```sql
SELECT * FROM students
WHERE Marks = (SELECT MAX(Marks) FROM students);
```

The inner query `(SELECT MAX(Marks) FROM students)` runs first and returns `91`, then the outer query finds all students with `Marks = 91`.

### Find students scoring above average

```sql
SELECT Name, Marks FROM students
WHERE Marks > (SELECT AVG(Marks) FROM students);
```

### Subquery with IN

```sql
SELECT * FROM students
WHERE Address IN (
    SELECT Address FROM students
    GROUP BY Address
    HAVING COUNT(*) > 1
);
```

Returns students who live in a city that has more than one student.

## SELECT Execution Order

SQL does **not** execute in the order you write it. The actual execution order is:

```
1. FROM        -- Which table(s)
2. WHERE       -- Filter individual rows
3. GROUP BY    -- Group the remaining rows
4. HAVING      -- Filter groups
5. SELECT      -- Choose columns / compute expressions
6. DISTINCT    -- Remove duplicate results
7. ORDER BY    -- Sort the output
8. LIMIT       -- Restrict number of rows returned
```

This is why you **cannot** use a column alias from `SELECT` inside `WHERE` (because `WHERE` runs before `SELECT`), but you **can** use it in `ORDER BY` (which runs after `SELECT`).

## Complete Query Template

```sql
SELECT [DISTINCT] columns / expressions / aggregates
FROM table_name
WHERE condition           -- filter rows
GROUP BY column           -- group rows
HAVING condition          -- filter groups
ORDER BY column [ASC|DESC]
LIMIT count [OFFSET skip];
```
