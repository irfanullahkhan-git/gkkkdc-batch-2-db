## 1. DISTINCT - Remove Duplicates

`DISTINCT` returns only unique values, removing repeated entries.

```sql
SELECT DISTINCT Address FROM students;
```

| Address |
|---|
| Rawalpindi |
| Lahore |
| Islamabad |
| Peshawar |
| Nowshera |

There are 9 students but only 5 unique cities. `DISTINCT` removed the duplicates (Rawalpindi, Lahore, and Islamabad each appeared more than once).

## 2. LIMIT & OFFSET - Pagination

### LIMIT - Get first N rows

```sql
SELECT * FROM students LIMIT 2;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |

Returns only the **first 2** rows from the table.

### LIMIT with OFFSET - Skip then take

`OFFSET` tells how many rows to **skip** before starting to return.

```sql
SELECT * FROM students LIMIT 2 OFFSET 2;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |

Skips the first 2 rows (Asma, Bisma), then returns the next 2 (Bushra, Bilal).

```sql
SELECT * FROM students LIMIT 2 OFFSET 1;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |

Skips 1 row (Asma), then returns the next 2.

```sql
SELECT * FROM students LIMIT 3 OFFSET 4;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 6 | Ali | 22 | Peshawar | 91 |
| 7 | Gul | 24 | Nowshera | NULL |

Skips first 4 rows, then returns 3.

### OFFSET alone does NOT work (error)

```sql
SELECT * FROM students OFFSET 1;
```

```
ERROR 1064 (42000): You have an error in your SQL syntax
```

`OFFSET` **cannot** be used without `LIMIT`. You must always provide `LIMIT` first.

### Negative LIMIT does NOT work (error)

```sql
SELECT * FROM students LIMIT -3;
```

```
ERROR 1064 (42000): You have an error in your SQL syntax
```

`LIMIT` only accepts **zero or positive** integers. You cannot use negative numbers.

## 3. LIKE with `%` Wildcard - Pattern Matching

`%` matches **zero or more** characters.

### Names starting with "B"

```sql
SELECT * FROM students WHERE Name LIKE "B%";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |

`"B%"` means: starts with B, followed by anything.

### Names starting with "Bi"

```sql
SELECT * FROM students WHERE Name LIKE "Bi%";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 4 | Bilal | 23 | Islamabad | 65 |

More specific — must start with "Bi".

### Names containing "i" anywhere

```sql
SELECT * FROM students WHERE Name LIKE "%i%";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 6 | Ali | 22 | Peshawar | 91 |
| 8 | Hina | 20 | Lahore | 85 |
| 9 | Iqra | NULL | Islamabad | 55 |

`"%i%"` means: anything, then "i", then anything. The letter "i" can appear at the start, middle, or end.

### Names ending with "l"

```sql
SELECT * FROM students WHERE Name LIKE "%l";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 7 | Gul | 24 | Nowshera | NULL |

`"%l"` means: anything followed by "l" at the end.

### Combining LIKE with OR - starts or ends with "i"

```sql
SELECT * FROM students WHERE Name LIKE "%i" OR Name LIKE "i%";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 6 | Ali | 22 | Peshawar | 91 |
| 9 | Iqra | NULL | Islamabad | 55 |

Ali ends with "i", Iqra starts with "I". (LIKE is case-insensitive in MySQL/MariaDB.)

### NOT with LIKE - names that do NOT start or end with "i"

```sql
SELECT * FROM students WHERE NOT (Name LIKE "%i" OR Name LIKE "i%");
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 7 | Gul | 24 | Nowshera | NULL |
| 8 | Hina | 20 | Lahore | 85 |

`NOT (...)` flips the entire condition — excludes Ali and Iqra.

### Complex: contains "i" but NOT at the start or end

```sql
SELECT * FROM students WHERE Name LIKE "%i%" AND NOT (Name LIKE "%i" OR Name LIKE "i%");
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 8 | Hina | 20 | Lahore | 85 |

The name must have "i" somewhere inside (`%i%`), but **not** as the first or last character. This filters out Ali (ends with i) and Iqra (starts with I).

## 4. LIKE with `_` Wildcard - Exact Character Count

`_` (underscore) matches **exactly one** character.

### Names that are exactly 3 characters long

```sql
SELECT * FROM students WHERE Name LIKE "___";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 6 | Ali | 22 | Peshawar | 91 |
| 7 | Gul | 24 | Nowshera | NULL |

Three underscores = exactly 3 characters. Only "Ali" and "Gul" match.

### NOT LIKE with `_` - names that are NOT 3 characters long

```sql
SELECT * FROM students WHERE Name NOT LIKE "___";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 8 | Hina | 20 | Lahore | 85 |
| 9 | Iqra | NULL | Islamabad | 55 |

Everyone except Ali and Gul (who have 3-character names).

## 5. BETWEEN - Range Filtering

`BETWEEN` checks if a value is within a range. It is **inclusive** on both ends.

### Numeric range: Age between 20 and 22

```sql
SELECT * FROM students WHERE Age BETWEEN 20 AND 22;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 6 | Ali | 22 | Peshawar | 91 |
| 8 | Hina | 20 | Lahore | 85 |

Returns students where `Age >= 20 AND Age <= 22`. Iqra (NULL age) and Gul (age 24) and Bilal (age 23) are excluded.

### Alphabetical range: Names between "A" and "D"

```sql
SELECT * FROM students WHERE Name BETWEEN "A" AND "D";
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 6 | Ali | 22 | Peshawar | 91 |

Returns names that fall alphabetically between "A" and "D" (exclusive of "D" itself since no name equals exactly "D"). Daniyal starts with "D" but "Daniyal" > "D" alphabetically, so it is excluded. Names starting with G, H, I are also excluded.

## 6. IN / NOT IN - Match Against a List

`IN` checks if a value matches **any** item in a given list. It replaces multiple `OR` conditions.

### The OR way (longer)

```sql
SELECT * FROM students WHERE Age = 20 OR Age = 21 OR Age = 23;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 8 | Hina | 20 | Lahore | 85 |

### The IN way (cleaner, same result)

```sql
SELECT * FROM students WHERE Age IN (20, 21, 23);
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 8 | Hina | 20 | Lahore | 85 |

Exact same result, but much cleaner to write and read.

### NOT IN - exclude specific values

```sql
SELECT * FROM students WHERE Age NOT IN (20, 21, 23);
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 2 | Bisma | 22 | Lahore | 72 |
| 6 | Ali | 22 | Peshawar | 91 |
| 7 | Gul | 24 | Nowshera | NULL |

Returns students whose age is **not** 20, 21, or 23. Notice Iqra (NULL age) is **not included** — `NOT IN` does not match NULL values.

## 7. IS NULL / IS NOT NULL

You **cannot** use `= NULL` or `!= NULL`. You must use `IS NULL` or `IS NOT NULL`.

### Find rows where Marks is NULL

```sql
SELECT * FROM students WHERE Marks IS NULL;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 7 | Gul | 24 | Nowshera | NULL |

Only Gul has no marks recorded.

### Find rows where Marks is NOT NULL

```sql
SELECT * FROM students WHERE Marks IS NOT NULL;
```

| ID | Name | Age | Address | Marks |
|---|---|---|---|---|
| 1 | Asma | 20 | Rawalpindi | 85 |
| 2 | Bisma | 22 | Lahore | 72 |
| 3 | Bushra | 21 | Islamabad | 91 |
| 4 | Bilal | 23 | Islamabad | 65 |
| 5 | Daniyal | 20 | Rawalpindi | 78 |
| 6 | Ali | 22 | Peshawar | 91 |
| 8 | Hina | 20 | Lahore | 85 |
| 9 | Iqra | NULL | Islamabad | 55 |

All 8 students who have marks recorded. Iqra has NULL age but her marks (55) are not NULL, so she appears.

## Summary

| Concept | Syntax | What it does |
|---|---|---|
| **DISTINCT** | `SELECT DISTINCT col` | Removes duplicate values |
| **LIMIT** | `LIMIT n` | Returns first n rows |
| **OFFSET** | `LIMIT n OFFSET s` | Skips s rows, returns next n |
| **LIKE %** | `WHERE col LIKE "A%"` | `%` matches zero or more characters |
| **LIKE _** | `WHERE col LIKE "___"` | `_` matches exactly one character |
| **NOT LIKE** | `WHERE col NOT LIKE "B%"` | Inverts the pattern match |
| **BETWEEN** | `WHERE col BETWEEN a AND b` | Inclusive range check (works on numbers and text) |
| **IN** | `WHERE col IN (1, 2, 3)` | Matches any value in the list |
| **NOT IN** | `WHERE col NOT IN (1, 2)` | Excludes values in the list (ignores NULLs) |
| **IS NULL** | `WHERE col IS NULL` | Checks for missing/unknown values |
| **IS NOT NULL** | `WHERE col IS NOT NULL` | Checks that value exists |

### Errors Encountered

| Query | Error | Reason |
|---|---|---|
| `SELECT * FROM students OFFSET 1` | Syntax error | `OFFSET` cannot be used without `LIMIT` |
| `SELECT * FROM students LIMIT -3` | Syntax error | `LIMIT` only accepts zero or positive integers |
