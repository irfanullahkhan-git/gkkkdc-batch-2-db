# ALTER TABLE, Multiple Tables, and JOINs

> Continues from [9.md](9.md). Introduces a second table (`departments`) and multi-table queries.

---

## 1. ALTER TABLE — Modifying an Existing Table

### Add a column

```sql
ALTER TABLE students ADD COLUMN department VARCHAR(20);
```

The new column is added with `NULL` in every existing row.

### Drop a column

```sql
ALTER TABLE students DROP COLUMN department;
```

Permanently removes the column and all its data.

> `ALTER TABLE` without specifying the table name is a syntax error:
> `ALTER TABLE ADD COLUMN …` → error

---

## 2. Populating a New Column

```sql
-- Set department for specific rows
UPDATE students SET department = "C Science" WHERE ID IN (1, 2, 4, 5, 7);

-- Fill the remaining NULLs
UPDATE students SET department = "Econ" WHERE department IS NULL;

-- Rename a value across multiple rows
UPDATE students SET department = "Computer Science" WHERE department = "C Science";
```

---

## 3. Creating a Lookup Table

```sql
CREATE TABLE departments (
    ID   INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(20)
);
```

> `AUTO INCREMENT` (with a space) is a syntax error — the correct keyword is `AUTO_INCREMENT`.

```sql
-- Insert multiple rows (one value per row)
INSERT INTO departments (name) VALUES ("Computer Science"), ("Economics");
```

| ID | name |
|---|---|
| 1 | Computer Science |
| 2 | Economics |

> `INSERT INTO departments (name) VALUES ("Computer Science", "Economics")` is an error — it tries to insert two values into one column. Use separate `()` groups.

## 4. Adding a Foreign Key Column

```sql
ALTER TABLE students ADD COLUMN department_id INT;

-- Link existing rows to the new lookup table
UPDATE students SET department_id = 1 WHERE ID IN (1, 2, 4, 5, 7);
UPDATE students SET department_id = 2 WHERE department_id IS NULL;

-- Remove the redundant text column
ALTER TABLE students DROP COLUMN department;
```

## 5. JOIN — Combining Two Tables

A `JOIN` combines rows from two tables based on a matching condition.

### (INNER) JOIN — Only Matching Rows

```sql
SELECT *
FROM students stu
JOIN departments dept ON stu.department_id = dept.ID;
```

Returns only rows where `department_id` has a match in `departments`. Samghan (NULL department_id) and Habib (department_id = 3, no matching department) are excluded.

### Selecting specific columns with aliases

When two tables have columns with the same name (both have `name`), qualify with the table alias:

```sql
SELECT stu.Name AS "Student Name", dept.name AS "Department Name"
FROM students stu
JOIN departments dept ON stu.department_id = dept.ID;
```

| Student Name | Department Name |
|---|---|
| Asma | Computer Science |
| Bisma | Computer Science |
| Bushra | Economics |
| Bilal | Computer Science |
| Daniyal | Computer Science |
| Ali | Economics |
| Gul | Computer Science |
| Hina | Economics |
| Iqra | Economics |

> Using an unqualified ambiguous column causes an error:
> `SELECT name …` → `Column 'name' in field list is ambiguous`

### INNER JOIN (explicit keyword — same result)

```sql
SELECT stu.Name AS "Student Name", dept.name AS "Department Name"
FROM students stu
INNER JOIN departments dept ON stu.department_id = dept.ID;
```

`JOIN` and `INNER JOIN` are identical in MariaDB/MySQL.

### LEFT JOIN — All Left Rows + Matching Right Rows

```sql
SELECT stu.Name AS "Student Name", dept.name AS "Department Name"
FROM students stu
LEFT JOIN departments dept ON stu.department_id = dept.ID;
```

Returns **all students** regardless of whether they have a matching department. Students without a match (Samghan, Habib) appear with `NULL` for the department name.

| Student Name | Department Name |
|---|---|
| Asma | Computer Science |
| … | … |
| Samghan | NULL |
| Habib | NULL |

---

## 6. Types of JOINs Summary

| JOIN Type | Returns |
|---|---|
| `(INNER) JOIN` | Only rows with a match in both tables |
| `LEFT JOIN` | All rows from the left table; NULLs for unmatched right rows |
| `RIGHT JOIN` | All rows from the right table; NULLs for unmatched left rows |
| `FULL OUTER JOIN` | All rows from both tables (not natively supported in MariaDB — use UNION) |

---

## Key Points

| Concept | Notes |
|---|---|
| `ALTER TABLE t ADD COLUMN` | Adds a column (existing rows get NULL) |
| `ALTER TABLE t DROP COLUMN` | Removes a column permanently |
| `AUTO_INCREMENT` | Must be used with a key (PRIMARY KEY or UNIQUE) |
| Table alias | `FROM students stu` — use `stu.col` to qualify column names |
| `INNER JOIN` | Excludes rows with no match |
| `LEFT JOIN` | Includes all left-table rows; unmatched right side is NULL |
| Ambiguous column | Qualify with alias: `stu.name`, not just `name` |
