# Introduction to MariaDB ‚Äî Databases, Tables, and Basic DML

> First session. Covers connecting to MariaDB, exploring existing databases, and performing basic DDL and DML operations.

---

## Connecting to MariaDB

```
mysql -u root -p
```

On success you get the MariaDB monitor prompt. Commands end with `;` or `\g`.

---

## Exploring Existing Databases

```sql
SHOW DATABASES;
```

Lists all databases on the server (e.g. `college`, `information_schema`, `mysql`, etc.).

---

## Creating and Selecting a Database

```sql
DROP DATABASE college;   -- remove if it already exists
CREATE DATABASE college;
USE college;
```

`USE` sets the active database for all subsequent queries.

---

## Creating a Table

```sql
CREATE TABLE students (
    ID      INT,
    Name    VARCHAR(20),
    Address TEXT
);
```

- `INT` ‚Äî integer numbers
- `VARCHAR(n)` ‚Äî variable-length string up to *n* characters
- `TEXT` ‚Äî long text with no fixed length limit

### Inspecting the structure

```sql
DESC students;
```

| Field | Type | Null | Key | Default | Extra |
|---|---|---|---|---|---|
| ID | int(11) | YES | | NULL | |
| Name | varchar(20) | YES | | NULL | |
| Address | text | YES | | NULL | |

---

## INSERT ‚Äî Adding Rows

### Single row with all columns specified

```sql
INSERT INTO students (ID, Name, Address) VALUES (1, "Ali", "Nowshera");
```

### Multiple rows in one statement

```sql
INSERT INTO students (ID, Name, Address) VALUES
    (2, "Aslam", "Nowshera"),
    (3, "Akber", "Peshawar");
```

### Omitting optional columns (they become NULL)

```sql
INSERT INTO students (ID, Name) VALUES (4, "Ayaz"), (5, "Aiza");
```

Rows 4 and 5 will have `NULL` in the `Address` column.

### Using `VALUE` shorthand (all columns, positional)

```sql
INSERT INTO students VALUE (6, "Baseer", "Rawalpindi"), (7, "Bushra", "Islamabad");
```

`VALUE` is an alias for `VALUES` and requires values for every column in order.

## SELECT ‚Äî Querying Data

```sql
SELECT * FROM students;
```

| ID | Name | Address |
|---|---|---|
| 1 | Ali | Nowshera |
| 2 | Aslam | Nowshera |
| 3 | Akber | Peshawar |
| 4 | Ayaz | NULL |
| 5 | Aiza | NULL |
| 6 | Baseer | Rawalpindi |
| 7 | Bushra | Islamabad |

`*` means "all columns".

---

## NULL Values

When a column value is not provided during `INSERT`, it defaults to `NULL`.  
`NULL` means *unknown* or *missing* ‚Äî it is **not** the same as zero or an empty string.

---

## Key Points

| Concept | Notes |
|---|---|
| `CREATE DATABASE` | Creates a new database |
| `USE` | Switches the active database |
| `CREATE TABLE` | Defines table structure and column types |
| `DESC` | Shows column definitions of a table |
| `INSERT INTO ‚Ä¶ VALUES` | Adds one or more rows |
| `SELECT *` | Returns all rows and columns |
| `NULL` | Represents a missing / unknown value |
